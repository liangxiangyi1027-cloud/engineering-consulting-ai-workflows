# 工程咨询 AI 工作流

本仓库用于沉淀“区域发展与全过程工程咨询”场景下的 AI 工作流、业务规则、标准模板、项目案例和自动化工具。

它不是单纯的软件代码仓库，而是一个面向工程咨询业务的长期知识与流程资产库，重点服务于：

- 投资估算表自动检查
- 可研、初设、概算一致性校核
- 设计院修改意见与通知单生成
- 会议纪要、责任清单、项目推进表整理
- 专项债、中央预算内等政策资金申报逻辑梳理
- Word、Excel、PDF 等工程资料自动化处理

## 目录说明

| 目录 | 用途 |
| --- | --- |
| `00_使用说明` | 仓库使用方法、AI 协作规则、文件命名规范 |
| `01_业务知识库` | 按业务板块沉淀咨询方法、经验、典型问题 |
| `02_政策与规范库` | 政策、规范、申报要求、审查口径 |
| `03_标准模板库` | 可研、初设、概算、专项债、会议纪要等模板 |
| `04_项目工作流` | 各类项目从输入到输出的标准处理流程 |
| `05_AI提示词库` | 可复用的 AI 提示词、审查指令和生成指令 |
| `06_自动化工具` | 后续逐步开发的 Excel、Word、PDF、报告生成工具 |
| `07_项目案例库` | 典型项目资料、过程文件、成果和复盘 |
| `08_数据字典与规则库` | 投资科目、建设内容、专项债规则、一致性规则 |
| `09_输出成果库` | AI 或人工整理后的标准化成果 |
| `99_归档` | 已完成项目、旧版模板、临时资料归档 |

## 建设原则

1. 先沉淀业务规则，再开发自动化工具。
2. 每个项目都保留“输入资料、处理过程、输出成果、复盘记录”。
3. AI 输出必须能追溯到原始资料、政策依据或人工判断。
4. 模板、规则和提示词要持续迭代，不追求一次成型。
5. 涉密、敏感或未脱敏项目资料不直接提交到公开 GitHub 仓库。

## 推荐使用方式

新项目开始时，先在 `07_项目案例库` 中建立项目文件夹；处理过程中将可复用规则沉淀到 `08_数据字典与规则库`，将成熟提示词放入 `05_AI提示词库`，将最终成果放入 `09_输出成果库`。

当某类工作流稳定后，再进入 `06_自动化工具` 开发对应脚本或小工具。
